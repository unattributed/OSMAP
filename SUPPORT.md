# Support

## Getting Help

OSMAP is still an actively developed prototype. The best support path depends on
the type of question.

Use a GitHub issue for:

- reproducible bugs
- documentation problems
- narrowly scoped feature proposals that fit the existing project goals

Use private email for:

- security reports
- sensitive operational information that should not be posted publicly

See [`SECURITY.md`](SECURITY.md) for private security reporting guidance.

## Before Opening an Issue

Please check:

- [`README.md`](README.md)
- [`docs/README.md`](docs/README.md)
- [`docs/KNOWN_LIMITATIONS.md`](docs/KNOWN_LIMITATIONS.md)
- existing issues and pull requests

## What Maintainers Need

Helpful reports usually include:

- the exact OSMAP commit or branch
- operating system and version
- whether the problem was reproduced on OpenBSD, QEMU, or another environment
- steps to reproduce
- expected behavior
- actual behavior
- bounded logs or error output with secrets removed
